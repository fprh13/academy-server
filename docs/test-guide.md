# Testing Guide

## 목적

이 문서는 이 저장소의 검증 실행 규칙을 정의한다.

모든 검증은 자동화 가능한 방식으로 수행되어야 한다.

---

## 금지 사항

You MUST NOT:

* 테스트를 비활성화한다
* assertion을 약화한다
* 설정을 제거하여 우회한다
* mock / hardcode로 임시 통과시킨다
* 원인과 무관한 변경을 수행한다

---

## 기대 상태

작업 종료 시 반드시 다음 상태여야 한다:

* 모든 검증이 통과된 상태
* 실패를 숨기지 않은 상태
* 변경이 재현 가능한 상태
* 동일 명령으로 동일 결과가 보장되는 상태
